# ADR 0001: Start with documentation

## Status

Accepted

## Context

Репозиторий пустой. Нужно учебное личное веб-приложение — трекер достижения цели на стыке пути Ника из «Землянина» и проверенных методик (Goal Pyramid, OKR, SMART, Atomic Habits, WOOP). Есть риск сразу уйти в выбор стека и UI без согласованной доменной модели.

## Decision

Начинаем **docs-first**:

1. Зафиксировать vision, methodology, glossary, MVP requirements, data model, backlog.
2. Код и scaffold приложения — только после ревью этих документов.
3. Технические умолчания (Next.js, TypeScript, локальное хранение, русский UI) записаны как намерение, но не реализуются в этой итерации.

## Consequences

- Плюс: общий язык и границы MVP до написания кода.
- Плюс: легче менять модель Ника/методик текстом, пока нет миграций.
- Минус: нельзя «пощупать» UI сразу; следующий шаг — scaffold после утверждения docs.
