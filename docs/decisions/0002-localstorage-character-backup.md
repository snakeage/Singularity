# ADR 0002: localStorage + JSON character backup

## Status

Accepted

## Context

Нужно хранить данные личного трекера без сервера и регистрации. Важно отличать бэкап **персонажа** (мечты, этапы, отметки) от бэкапа **кода проекта** (git).

## Decision

1. Живые данные MVP хранятся в браузере в `localStorage` под ключом `singularity.appData.v1`.
2. Экспорт — JSON-файл вида `singularity-character-YYYY-MM-DD.json` с обёрткой:
   - `kind: "singularity-character-backup"`
   - `exportedAt`
   - `data` (AppData v1)
3. Импорт полностью **заменяет** текущие данные после подтверждения пользователя.
4. Полноценный SQLite-файл на диске и облачная синхронизация — вне MVP.

## Consequences

- Плюс: быстрый старт, нет бэкенда.
- Плюс: пользователь понимает бэкап как «сейв персонажа».
- Минус: данные привязаны к браузеру/профилю; очистка сайта стирает прогресс без файла бэкапа.
- Минус: нет авто-синка между устройствами.
