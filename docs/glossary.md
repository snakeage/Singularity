# Glossary

Единый язык продукта. В UI и в коде стараемся использовать эти термины.

| Термин | Англ. (код) | Определение |
| --- | --- | --- |
| Мечта | `Dream` | Глобальная цель: что, зачем, образ результата, горизонт. Далёкий ориентир, не ежедневный чеклист. |
| Точка А / «Где я сейчас» | `PointA` | Честный старт: что умею, чем располагаю, что мешает. В UI — «Где я сейчас»; «ограничения» = условия, которые держат на месте (не оправдания). |
| Контекст | `Dream.context` | Правила среды вокруг мечты: ограничения, условия, «как устроен мир» для этой цели. |
| Этап | `Stage` | Ступень лестницы к мечте. У этапа есть цель (objective) и критерии выхода. Активен один этап. |
| Рубеж | `Milestone` | **Доказательство роста** на этапе («экзамен»), не ежедневное дело и **не препятствие**. Закрыл рубежи → готов к следующему этапу. |
| Критерии выхода | `exitCriteria` | Definition of done этапа: что должно быть правдой, чтобы сказать «вырос». |
| Практика | `Practice` | Повторяемое действие (ежедневно/еженедельно), закалка текущего этапа. |
| Отметка | `CheckIn` | Факт выполнения практики или продвижения по рубежу в конкретный день. |
| Источник роста | `GrowthSource` | Книга, курс, ментор, тренажёр, **ИИ** — «учитель/знание» на пути. |
| Прокачка из сети | — | Мотив из книги: ускоренное обучение через сетевые технологии. В продукте — ИИ как ускоритель + слой «Сеть» (см. [mythology-network.md](./mythology-network.md)). |
| База | состояние `Practice` в сети | Компетенция: установлена (первая норма) → закалка (повторы) → апгрейд (рост min / фокуса) → подтверждение (рубеж/учитель). |
| Сеть | UI-слой / уровень канала | Интерфейс пути; уровень сети ≈ уровень персонажа, без валюты и военных веток. |
| Капсула | сессия таймера | Фокус-сессия практики; арт и ощущение «база качается», не отдельная экономика. |
| Курс / не отклоняться | — | Принцип: двигаться к мечте и отсекать боковые квесты, которые не усиливают текущий этап. |
| Окно обучения | `learningWindow` | Появилось время/энергия — вложить в учёбу под этап, а не сжечь впустую. |
| Препятствие | `Obstacle` | Корень ветки: что сбивает с пути. |
| План «если → то» | `ImplementationIntention` | Ветка ответа на препятствие (`obstacleId`). На одно препятствие — несколько планов. |
| Обзор | `Review` | Регулярная рефлексия (в MVP — еженедельная): что сработало, что мешало, что меняем; были ли окна на обучение. |
| Лестница | — | Метафора пути: Мечта → Этапы → Рубежи/Практики. |
| Активный этап | `active Stage` | Единственный этап, на котором сейчас идёт основная работа. |

## Статусы (черновик)

**Dream:** `draft` → `active` → `paused` → `achieved` | `abandoned`

**Stage:** `planned` → `active` → `completed` | `skipped` (skipped — осторожно, против духа метода; в MVP лучше не поощрять)

**Milestone:** `open` → `done`

**Practice:** `active` | `paused` | `retired`
